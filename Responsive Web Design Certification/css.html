CSS!

You can add styles with CSS in HTML tags with the style attribute. This is called inline CSS.

You can also add styles with a style block.

The most popular way to style is with external files. You have to link the external page for this to work.

I have added code between [ ] on this page, when using the code omit the brackets. I have also not included all the ways to style here, there are far to many! check out devdocs.io for more specific information.

CSS Selectors

When you are styling something with the style block / elements or an external file you have to select the thing you wish to apply styles too. CSS allows for a few types of selectors:

  1. Element
    • To set styles for every element of the same type ( all h2's) on a page you can use the elements tag (h2, p, img ) the element as the selector.
  2. Classes
    • To use classes place [ class="class-name" ] within the opening tag of the element to be styled, then style with your preferred method.
    • To style use [.class-name {styles} ]
    • Elements can have as many classes as needed to get the look you are going for.
    • Classes can be placed on as many elements as needed to get the look you are going for.
  3. Id's
    • To use id's place [ id="id-name" ] within the opening tag of the element to be styled, then style with your preferred method.
    • To style use [ #id-name {styles} ]
    • Must be unique! Basically one use only.
    • Each element can have only one id.
    • Id's allow you to select and manipulate elements
  4. Attributes
    • Styles are applied to elements with the matching attribute/value
    • To style use [ attr=value {styles} ]
    • Has many additional qualifier symbols

CSS Specificity

  • The way you select elements to be styled set a level or priority that be overwritten or overwrite others.
  • Where you place your styles matters as well. Styles are read top to bottom with things being nearer the top being overwritten by styles lower in the document.
  • If you include !important in your declaration it will force it and over ride everything.
  • How you select your elements can effect how things are applied to them.
  • Inline styles > Id's > Classes, attributes and pseudo-classes > Elements and pseudo-elements

You can import fonts from google by adding a link elements with [href="https://fonts.googleapis.com/css?family=font-name" rel="stylesheet" type="text/css"] and setting the font-family property. If you do this it is best to create a fall back so that if the external font is not reachable you still have some control over the fonts. [font-family: FAMILY_NAME, GENERIC_NAME;] These generic font families include monospace, serif and sans-serif.

Box Model

HTML treats everything as boxes. We can control the way these boxes look by changing the properties of the box model:

  1. Margin - The outer most box, is whitespace between the elements. Styles with margin properties
  2. Border- Space for outlining the content and padding. Styled with border properties
  3. Padding- Whitespace between the content and border. Styled with padding properties.
  4. Content-Where your content is shown. Sized with content properties as well as hight and width for size control.

With some of these you can use top, right, bottom, left properties or place them all in a single line [margin: 10px 20px 10px 20px;], or even [margin: 10px 20px]. In the second example the values are for top/bottom and right/left

Common Length Units

You use length units to size elements. The types of length units are:

  1. Absolute- Not recommended, especially for font sizing. Users are unable to change these values with zoom functions causing negative usability.
    • px - One pixel.
  2. Relative
    • Font-relative lengths
      • em - Represents the calculated font-size of the element. If used on the font-size property itself, it represents the inherited font-size of the element.
      • rem - Represents the font-size of the root element (typically ). When used within the root element font-size, it represents its initial value (a common browser default is 16px, but user-defined preferences may modify this).
    • Viewport-percentage lengths
      • vh - Equal to 1% of the height of the viewport's initial containing block.
      • vw - Equal to 1% of the width of the viewport's initial containing block.

CSS Variables

You can add variables to css to make styling faster. First you have to declare the variable in your style document. with [ --var-name: value; ]. You can then use them like: background-color: var(--var-name, fallback). It is also smart to place a fallbacks right before variables, this helps catch browser compatibility issues. Variables are only available withing the selector that they are declared, placing them in a :root selector will have them globally accessible. Variables have the same top to bottom rules as other declarations.
