Basic HTML and HTML5

Tag Anatomy

HTML is compose of tags. Browsers use these tags to format the structure of a webpage and display content. Most of the tags require both opening and closing tags. There are a handful of self closing tags. These tags are used to define Elements. 

                              <opening tag> Content <closing tag>

Several things can go in the opening tag, but the element name is mandatory and should be the first thing in the tag. Opening tags can also include attributes, some elements even require attributes. Closing tags hold the element name preceded by a slash (/). Below is an example of a tag with some attributes. This tag will place a link on the page, with a class, id and a title. Because of the target attribute it will open uin a new tap or window. 

                              <a href="https://www.google.com/" class"my-class" id="my-id" title="Link to google search" target="_blank"> Content </a>

A Multitude of Elements

HTML has a ton of elements to help structure a page. Some of these tags are on every HTML page and some are rarely used. A list of every tag (I think) can be found on DevDocs.io. Convention is to keep elements lowercase.

Tags that are on every page don't directly do anything to the page content, they provide data to help other things structure the page, import styles and scripts, and provide meta information. These type of tags live in the head section of the page. If you look at the boilerplate above you can see some examples. More information about the head section and what it can house can be found at MDN Web Docs.

Other elements are "fancy containers". These are called semantic elements, they have a meaning and hold certain type of data. These elements help with search engine optimization, other read your code, as well as accessability. A few semantic elements are:

< header >
  • Contains intro information such as logos, section titles and navigation elements
  • Can have many header sections on a page
  • Don't place within the footer, address or other header tags
< section >
  • Contains sections within the page
  • Sections are groupings of like content
  • Can have unlimited per page
  • Normally have a sub-title then content
< article >
  • Contains independent/self-contained content
  • Should be able to take this and have it stand on its own
  • Common uses are: forum posts, blog post and articles
< nav >
  • Holds navigation elements
  • Intended for blocks of links for navigating the site
< footer >
  • Normally placed at the bottom of a page
  • Holds repetitive information such as: copyright, navigation, contact and legal information.

Semantic elements are important, but they generally don't create anything we can see on the page. HTML has a ton of elements that do create things we can display on the page, too many to go over here. Some commons ones are:

< h1 >, < h2 >,< h3 >,< h4 >,< h5 >,< h6 >
  • These elements are used for header information
  • Avoid using heading tags to resize text. Instead, use the CSS font-size propert.
  • Avoid skipping heading levels: always start from h1, followed by h2 and so on.
  • One h1 per page, should be the main title of the page.
< a >
  • Creates a link out of the contents of the tags
  • Requires a href attribute
  • The target attribute allows you to specify where to open the link
< img >
  • used to display images
  • Requires a src attribute that points to the source of the image
  • Requires a alt attribute that provides a text alternative for display if the image can't be displayed for some reason (leave empty if image is only decorative)
  • Always give a height and width to img elements
< ol >, < ul >, < li >
  • These elements form lists
  • Ordered Lists are prefixed by numbers
  • Unordered List are prefixed by bullets
  • List items are placed withing the list type tags and contain the
< !-- > and < -- >
  • HTML comments
  • Anything between these tags are not displaied on the page
  • Useful for adding notes to code
  • Useful for troubleshooting with problems occur
< form >
  • Contains elements that create forms
  • Forms can be very complicated
  • For a list of elements that create forms see: w3schools HTML Form Elements page

Variables and Data types

JavaScript allows many types of data, but is a 'loosely typed' language that does not require you to tell the code what your data is when you declare it. Declaring a variable means that you are telling your program you have some data and what you want to call that data. For example if we needed to do something in your code based on age you might create (declare) a variable called age. There are many data types, you can see them at w3schools.com. 

                         keyword name = value;

Above is the anatomy of a variable declaration. The keyword can be one of var, let, or const. The name must start with a letter but can be any combination of letters, numbers, underscores and dollar signs. Names are also case sensitive. The equals sign is called the assignment operator in java script. You are assigning the value you set to the name you set. The value can be any number of things. The only thing to consider is if the value is a string (words) or a numerical value. Numerical values and strings don't mix, 5 + apple is not something you can do. Strings are encapsulated with single or double quotes.

Variable Manipulation

The whole point of having variables to store and use changeable data and of course we have several ways to do this! You can simply give the variable a new value buy reassigning it something new. This new value can even be another variable or the result of other parts of the program. 

                                   //creating variables with no values
var a;
var b;
var c; 

//adding values to the variables
a = a + 1;
b = b + 5;
c = c + " String!";

camelCase

Camel Case is the industry standard for naming variable in JavaScript. With this way f writing variables you start with a lowercase letter then capitalize every word in a multi-word variable. thisIsAveryLongVariableNameButItIsInCamleCase.

Dealing with Strings

As said before strings are encapsulated with quotes, but what if you need your string to have quotes? There are a few ways to do this. First you can escape the string. This is done by placing \ before each quote that needs to be escaped. This can be tricky and take up a ton of time if you need to escape a large about of quotes. An easier way is to encapsulate your string with single quotes, doing this allows you to use double quotes in the string. 

                              var sampleStr = "I am \"Escaping!\" The string."; 
Var singleQuote = 'I can use "as" many "quotes" as "I" want';

Quotes are not the only thing you can escape in strings. To be honest I hope to learn a better way to do this. If for some reason its is needed more information can be found over at w3schools.

Finally you can "add" strings together with the addition operators ( + and +=). This is called concatenation. You can also display variables via concatenation. 

                              var myName = "Catherinitude";
var myStr = 'My name is ' + myName +' and I am well!';

JavaScript allow for us to get information about data including strings. One of them is .length and it is used like so: 

                         myString.length

This will return an integer that is equal to the number of characters in the string (minus 1). This is because JavaScript is a Zero-based indexing language. This means that the first value is 0. .length works much the same with other things. some Examples: 

                         var myString= 'Hello from JavaScript';
myString.length will return 21

You can target specific parts of a string by adding brackets at the end and adding the position you wish to target. You can ask for .length-1 to get the last value. 

                    var myString= 'Hello from JavaScript';
myString.length[7]      // will return r
myString.[myString.length-1]      //  will return t
myString.[myString.length-4]      //  will return r

Functions

Functions allow for the use of code repeatably. This is useful so that you can repeat actions without having to retype code. Keeping repetition of code to a minimum is important. Concise code helps keep code easy to understand as well as optimized. If you need to do something more then once if belongs in a function! A function looks like this: 

                              function functionName(comma, separated, parameters){
// function code lives here for example:
return(comma + separated + parameters);
//This will add all three parameters values together.
}                            
functionName(1, 5, 4) // would return 10

Generally your function will need to return something this is done with return() method. Once the function returns something it stops working so only one return per function can be returned. Functions can be super simple or huge and complicated.

Scope

Scope is basically where variables can be seen / used. There are several levels of scope. The largest scope is global scope. Variables declared with the var keyword are created in global scope. Global scope lets everything interact with it and can have some issues. The other type os scope is local scope. There is a nice article at Scotch.io on what each scope is and how it is used.

If Statements

If statements are used to test data and do things based on the outcome of the tests. This testing is done with logical operators found above in Logical Operators Table. 

                         if (testing logic) {
     do this if testing logic is true
} 
     do this if testing logic is false
}

The above is the most basic if statements, they can be made more complicated like this: 

                         if (testing logic 1) {
if testing logic 1 is true do this
} else if (testing logic 2) {
if testing logic 2 is true do this
} else if (testing logic 3) {
if testing logic 3 is true do this
} else if (testing logic4) {
if testing logic 4 is true do this
} else {
if everything is false do this
}

It is important when working with longer statements to keep in mind that JavaScript reads and executes code in top down left right order. This means if you have a test for less than 25 followed by a test for less than 5 and your testing 4, code will be triggered at less than 25 and the less than 5 will never be tested.

Switches

Like If statements switch statements allow for logical manipulation of data by matching rules with code to run. In switch statements you use a default value to as a catch all. When something matched the case the code is run until it hits a break statement.

                         switch (testing logic) {
     case 0:
          code here is triggered when 0 is the result of the testing logic
          break;
     case "apple":
          code here is triggered when "apple" is the result of the testing logic
          break;
     case 32:
          code here is triggered when 0 is the result of the testing logic
          break;
     default :
          code here is triggered when no other cases are met that is why its at the end
          break;
     }
